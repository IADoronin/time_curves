# Кривые роста — визуализация и запись экспериментов

Два настольных приложения (PyQt6), работающих с одним форматом `.xlsx`:

- **`main.py`** — визуализатор: группирует эксперименты по условию, усредняет
  повторы (±SD), выравнивает сдвиг времени, экспортирует графики для статей.
- **`record.py`** — запись данных: настройка схемы эксперимента, ведение кривых
  в SQLite, экспорт в те же `.xlsx` (см. раздел «Запись данных» ниже).

## Визуализатор

Группирует эксперименты по выбранному условию и усредняет повторы, позволяя
включать/выключать отдельные кривые.

## Формат входных данных

Каждый эксперимент — отдельный `.xlsx` с двумя листами:

- **`meta`** — условия эксперимента, таблица из двух колонок `parameter` / `value`:

  | parameter    | value                     |
  |--------------|---------------------------|
  | sample_name  | aero_with_co2_malate_1    |
  | start_date   | 2026-05-12 10:00          |
  | aeration     | aerobic                   |
  | co2          | CO2+                      |
  | substrate    | malate                    |
  | replicate    | 1                         |

- **`data`** — временные ряды. Первая колонка — время, остальные — измеряемые
  величины (напр. `time_h`, `OD600`, `pH`, `substrate_mM`). Моменты измерений
  **нерегулярные** (интервалы ~1.3–3.2 ч, как при ручном снятии), число точек и
  конец эксперимента у образцов различаются.

## Запуск

```bash
conda activate growth_viz          # выделенное окружение проекта
python main.py                     # откроет test_data по умолчанию
python main.py /путь/к/папке       # или указать свою папку с .xlsx
```

В окне три колонки: слева — загрузка/выбор группировки и показателя, коррекция
сдвига времени, дерево образцов с чекбоксами; по центру — график; справа —
настройки оформления.

## Коррекция сдвига времени (выравнивание кривых)

Начальную точку («ноль») эксперимента часто определяют неточно, из-за чего
повторы сдвинуты друг относительно друга по времени и полоса ±SD раздувается на
фазе роста. Группа **«Коррекция сдвига времени»** (левая колонка) это исправляет:

- **Выравнивать кривые** — вкл/выкл.
- **Метод**:
  - *По уровню сигнала* — сдвиг оценивается по времени пересечения опорным
    показателем заданного уровня (линейная интерполяция). Устойчиво к разреженным
    данным — метод по умолчанию.
  - *По макс. скорости роста* — по времени максимальной |скорости| изменения.
- **Опорный** — по какому показателю считать сдвиг (сдвиг применяется ко всем
  величинам образца; по умолчанию OD600).
- **Уровень** — порог для метода «по уровню сигнала».

Выравнивание идёт **внутри каждой группы** (повторы приводятся к среднему
положению ориентира своей группы), поэтому реальные различия между группами
(напр. разный лаг у субстратов) сохраняются, а не стираются.

## Настройки оформления (правая колонка)

- **Точки** — маркеры (вкл/выкл, стиль, размер).
- **Линии** — толщина среднего, прозрачность повторов и полосы ±SD.
- **Шрифт** — семейство и размер.
- **Оси** — шаг X/Y (0 = авто), сетка.
- **Прочее** — легенда, заголовок.
- **Экспорт** — размер фигуры (см/дюйм = соотношение сторон), DPI, формат
  (PNG/PDF/SVG/TIFF/EPS), предпросмотр в реальных пропорциях, кнопка сохранения.

## Экспорт для статей

Задайте ширину и высоту (в см или дюймах — это и есть соотношение сторон) и DPI,
затем «Сохранить график…». Файл сохраняется **точно** в заданном размере
(напр. 8.9 см × 6 см @ 600 dpi), шрифты в файле совпадают с настройками на экране.
Векторные форматы (PDF/SVG/EPS) — для журналов, требующих масштабируемую графику.
Галка «Предпросмотр в этих пропорциях» показывает график ровно в тех же
пропорциях, что и будущий файл (letterbox).

## Тестовые данные

```bash
python generate_test_data.py       # (пере)создаёт test_data/ с 6 файлами
```

meta извлекается из имени файла по схеме `{aeration}_{co2}_{substrate}_{replicate}`.
Данные имитируют реальные: у каждого образца **сдвиг нулевой точки** (±~1.8 ч —
неточно определённый старт) и **нерегулярные интервалы** измерений.

## Запись данных (`record.py`)

Приложение для ручной записи кривых, выдающее совместимые с визуализатором
`.xlsx`. Хранилище — **SQLite** (один файл `.db` = один эксперимент), экспорт в
Excel — по запросу.

```bash
conda activate growth_viz
python record.py                 # спросит, какую базу .db создать/открыть
python record.py experiment.db   # или сразу указать базу
```

Порядок работы:

1. **Схема эксперимента** (меню «Эксперимент → Схема…»): задайте
   - *свойства-условия* (лист meta): имя + тип **enum** (список вариантов) или
     **numeric** (+ единица);
   - *измеряемые величины* (колонки листа data): имя + единица;
   - *единицу времени* (часы/минуты/сутки → колонка `time_h`/`time_min`/`time_d`).
2. **Новая кривая** — форма по схеме (enum → выпадающий список, numeric → поле с
   проверкой числа); имя подставляется автоматически из условий, старт = сейчас.
3. **Запись точек** — время вводится вручную (с проверкой типа) или кнопкой
   **«Сейчас»** (= прошедшее от старта в выбранных единицах) + значения величин;
   «Добавить точку». Можно вести несколько кривых, переключаясь в списке.
4. **Завершить кривую** / **Экспорт в Excel…** (одной кривой или всех).
5. **Графики** (кнопка или меню «Вид → Открыть графики») — открывает окно
   визуализатора прямо по текущим кривым (кривые экспортируются во вспомогательную
   папку `<база>_plots/` и загружаются). Повторное нажатие обновляет график.

Экспортированную папку можно также открыть визуализатором отдельно:
`python main.py <папка>`.

## Структура

```
growth_viz/
  model.py         — dataclass Sample (name, meta, data)
  loader.py        — загрузка .xlsx (листы meta/data), ключи группировки
  writer.py        — запись .xlsx (инверсия loader), общий модуль экспорта
  grouping.py      — группировка по условию + усреднение повторов (mean/std)
  alignment.py     — коррекция сдвига времени (выравнивание)
  plot_canvas.py   — холст matplotlib
  plot_settings.py, settings_panel.py — оформление и экспорт графиков
  main_window.py   — окно визуализатора
recorder/
  db.py            — слой SQLite (схема, кривые, точки) + экспорт кривых
  schema_dialog.py — настройка схемы эксперимента
  new_curve_dialog.py — создание кривой по схеме
  record_window.py — окно записи
main.py            — визуализатор (точка входа)
record.py          — запись (точка входа)
generate_test_data.py — генератор тестовых файлов
```
